# Contributing to Colab AI Coder\n\nThank you for your interest in contributing! This document provides guidelines for contributing to the project.\n\n## Getting Started\n\n### Prerequisites\n\n- Python 3.9+\n- Git\n- Docker (optional, for containerized testing)\n\n### Setup Development Environment\n\n```bash\n# Clone repository\ngit clone https://github.com/yourusername/colab-ai-coder.git\ncd colab-ai-coder\n\n# Create virtual environment\npython -m venv venv\nsource venv/bin/activate  # On Windows: venv\\\\Scripts\\\\activate\n\n# Install dependencies\ncd backend\npip install -e \".[dev]\"\n\n# Setup pre-commit hooks\npre-commit install\n```\n\n## Development Workflow\n\n### 1. Create Feature Branch\n\n```bash\ngit checkout -b feature/your-feature-name\n```\n\n### 2. Make Changes\n\n- Write code following the style guide\n- Add tests for new features\n- Update documentation\n\n### 3. Run Tests\n\n```bash\n# Run all tests\npytest tests/ -v\n\n# Run with coverage\npytest tests/ --cov=src --cov-report=html\n\n# Run specific test file\npytest tests/test_health.py -v\n```\n\n### 4. Lint & Format\n\n```bash\n# Format code\nblack src tests\nisort src tests\n\n# Check formatting\nruff check src tests\n\n# Type check\nmypy src\n\n# Security scan\nbandit -r src\n```\n\n### 5. Pre-commit Hooks\n\nHooks run automatically on `git commit`:\n- Format checks (black, isort)\n- Lint checks (ruff)\n- Type checks (mypy)\n- Security checks (bandit, semgrep)\n\nIf hook fails, fix and commit again.\n\n### 6. Push & Create Pull Request\n\n```bash\ngit push origin feature/your-feature-name\n```\n\nThen create PR on GitHub.\n\n## Code Style\n\n### Python\n\n- **Formatter**: Black (120 char line length)\n- **Linter**: Ruff\n- **Type Hints**: Required (PEP 484)\n- **Docstrings**: Google style\n\n### Example\n\n```python\n\"\"\"Module docstring.\"\"\"\n\nfrom typing import Optional\n\n\ndef generate_code(prompt: str, max_tokens: int = 256) -> str:\n    \"\"\"Generate code from prompt.\n    \n    Args:\n        prompt: Input prompt for code generation.\n        max_tokens: Maximum tokens to generate.\n        \n    Returns:\n        Generated code as string.\n        \n    Raises:\n        ValueError: If prompt is empty.\n    \"\"\"\n    if not prompt:\n        raise ValueError(\"Prompt cannot be empty\")\n    \n    # Implementation\n    return \"generated code\"\n```\n\n## Testing\n\n### Write Tests\n\n```python\ndef test_generate_endpoint(test_client: TestClient):\n    \"\"\"Test code generation endpoint.\"\"\"\n    payload = {\n        \"prompt\": \"def hello\",\n        \"task\": \"generate\",\n        \"max_tokens\": 100,\n    }\n    response = test_client.post(\"/api/v1/assistant/generate\", json=payload)\n    \n    assert response.status_code == 200\n    assert \"response\" in response.json()\n```\n\n### Coverage Target\n\n- Minimum 80% code coverage\n- All public APIs must have tests\n- Test both success and error cases\n\n## Git Commit Messages\n\nFollow conventional commits:\n\n```\n<type>(<scope>): <subject>\n\n<body>\n\n<footer>\n```\n\n### Examples\n\n```\nfeat(api): add streaming response support\n\nImplement WebSocket endpoint for real-time streaming\nof model responses.\n\nCloses #123\n```\n\n```\nfix(backend): handle OOM gracefully\n\nAdd fallback to smaller model when GPU memory is insufficient.\n```\n\n### Types\n\n- `feat`: New feature\n- `fix`: Bug fix\n- `docs`: Documentation\n- `style`: Code style (formatting, etc.)\n- `refactor`: Code refactoring\n- `perf`: Performance improvements\n- `test`: Test additions/changes\n- `ci`: CI/CD changes\n\n## Pull Request Guidelines\n\n### Before Submitting\n\n- [ ] All tests pass\n- [ ] Coverage is maintained (‚â•80%)\n- [ ] Code is formatted (black)\n- [ ] Lints pass (ruff)\n- [ ] Types check (mypy)\n- [ ] Security scan passes (bandit)\n- [ ] Documentation is updated\n- [ ] Commit messages follow convention\n\n### PR Description\n\n```markdown\n## Description\n\nBrief description of changes.\n\n## Type of Change\n\n- [ ] Bug fix\n- [ ] New feature\n- [ ] Breaking change\n- [ ] Documentation update\n\n## Related Issues\n\nCloses #123\n\n## Testing\n\nDescribe how you tested the changes.\n\n## Checklist\n\n- [ ] Tests added/updated\n- [ ] Documentation updated\n- [ ] No breaking changes\n```\n\n## Security\n\n### Reporting Vulnerabilities\n\n**Do NOT** open a public issue for security vulnerabilities.\n\nEmail: security@example.com\n\nInclude:\n- Description of vulnerability\n- Steps to reproduce\n- Potential impact\n\n## Community\n\n- **Discussions**: GitHub Discussions\n-
